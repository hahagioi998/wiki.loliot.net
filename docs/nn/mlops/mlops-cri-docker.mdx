## Installation

```yaml title="docker-installation.yaml"
---
- hosts: all
  become: yes
  tasks:
    - name: purge
      apt:
        name: '{{ item }}'
        state: absent
        update_cache: yes
      loop: ['docker', 'docker-engine', 'docker.io', 'containerd', 'runc']

    - name: install required
      apt:
        name: '{{ item }}'
        state: latest
        update_cache: yes
      loop:
        [
          'apt-transport-https',
          'ca-certificates',
          'curl',
          'gnupg',
          'lsb-release',
        ]

    - name: add docker GPG apt key
      apt_key:
        url: https://download.docker.com/linux/ubuntu/gpg
        keyring: /usr/share/keyrings/docker-archive-keyring.gpg

    - name: add docker repository
      apt_repository:
        repo: deb
          [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg]
          https://download.docker.com/linux/ubuntu
          {{ ansible_distribution_release }} stable
        filename: docker

    - name: install docker
      apt:
        name: '{{ item }}'
        state: latest
        update_cache: yes
      loop: ['docker-ce', 'docker-ce-cli', 'containerd.io']

    - name: set cgroupdriver to systemd
      copy:
        dest: '/etc/docker/daemon.json'
        content: |
          {
            "exec-opts": ["native.cgroupdriver=systemd"],
            "log-driver": "json-file",
            "log-opts": {
              "max-size": "100m"
            },
            "storage-driver": "overlay2"
          }

    - name: systemd enable docker
      ansible.builtin.systemd:
        name: '{{ item }}'
        enabled: yes
        state: started
      loop: ['docker', 'containerd']

    - name: systemd daemon-reload
      ansible.builtin.systemd:
        daemon_reload: yes
```

## Commands

### run

```shell
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

`-it` 옵션이 많이 사용됩니다.

- `--interactive`
- `--tty`

### pull, create, start, attach

`docker create` 또는 `docker run`을 실행했을 때, 이미지가 없다면 `docker pull` 명령어로 이미지를 가져옵니다.

`docker run -it` == `docker create -it` -> `docker start` -> `docker attach`

```shell
docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
```

```shell
docker start [OPTIONS] CONTAINER [CONTAINER...]
```

```shell
docker attach [OPTIONS] CONTAINER
```

### ps

```shell
docker ps [OPTIONS]
```

- `-a | --all`: 모든 컨테이너를 보여줍니다. 기본은 실행중인 컨테이너만 보여줍니다.

### inspect

```shell
docker inspect [OPTIONS] NAME|ID [NAME|ID...]
```

도커 오브젝트의 자세한 정보를 보여줍니다.

### stop

```shell
 docker stop [OPTIONS] CONTAINER [CONTAINER...]
```

### rm

```shell
docker rm [OPTIONS] CONTAINER [CONTAINER...]
```

### rmi

```shell
docker rmi [OPTIONS] IMAGE [IMAGE...]
```
